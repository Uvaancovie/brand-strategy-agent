### Phase 2: Enhance Firecrawl Search for Quantitative Data (1-2 hrs)

**Current Limitation:**
`firecrawl.service.ts` lines 172-178 use generic queries:
```typescript
const queries = [
  `${industry} market size ${country} 2025 2026 statistics`,
  `${industry} market trends growth rate ${country}`,
  // ...
];
```

These return blog posts and opinion pieces. Need **hard numbers**.

**Improvements:**

1. **Add quantitative keywords** to queries:
   - `"market size" OR "TAM" OR "Total Addressable Market" + "USD" OR "dollar"`
   - `"CAGR" OR "compound annual growth rate" + percentage`
   - `"industry report" OR "market research" + "2025"`
   - `"government statistics" OR "official data" + industry`
   - `"salary benchmark" OR "compensation survey" + country`
   - `"consumer spending" OR "household expenditure" + country`

2. **Prioritize source domains** (add scoring filter):
   - `.gov`, `.stats`, `.int` (UN, World Bank) — HIGH PRIORITY
   - `.com` with "report", "analysis", "study" in URL — MEDIUM
   - `.org` (industry associations) — MEDIUM
   - LinkedIn articles — LOW (often opinion-based)

3. **Enhance `buildResearchBrief()`** (lines 202-216):
   - Extract numbers/percentages from scraped content
   - Highlight sources that contain actual metrics (TAM, CAGR, market share %)
   - Tag sources by quality tier (gov > analyst > news > blog)
   - Format as: `[NUMBERS FOUND] Source title — key metric: $5.2B market size by 2026`

4. **Add query diversification per industry:**
   ```typescript
   const industrySpecificTerms = {

     'Healthcare': ['patient volume', 'hospital revenue', 'medical devices'],
     'E-commerce': ['online sales', 'conversion rate', 'average order value'],
   ```

1. **Stronger instruction following**: More precise JSON schema enforcement
2. **Better numeric reasoning**: Higher confidence on CAGR calculations

**Changes:**

- Include explicit currency formatting: `"Use ${currency} format. For South Africa: ZAR. For India: INR. For US: USD."`
