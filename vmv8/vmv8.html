01 — Mission

What VMV8 Does

VMV8 is an AI-powered reasoning system built to automatically extract, analyse, and generate comprehensive brand strategies from raw meeting conversations. It listens to audio, transcribes it, then applies the Voice Matrix 8-section framework to produce a complete Brand Identity Guiding Document (B.I.G Doc) — automatically.

JS — services/groq.service.js 
// Groq service — handles all AI calls with retry + context trimming
import Groq from 'groq-sdk';

const groq = new Groq({
  apiKey: import.meta.env.VITE_GROQ_API_KEY,
  dangerouslyAllowBrowser: true
});

const MAX_HISTORY = 20; // Prevent context window overflow

export async function extractBrandStrategy(transcript, history = []) {
  const trimmed = history.slice(-MAX_HISTORY);

  const response = await groq.chat.completions.create({
    model: 'llama-3.3-70b-versatile',
    response_format: { type: 'json_object' }, // Structured output — no regex
    messages: [
      { role: 'system', content: BRANDY_SYSTEM_PROMPT },
      ...trimmed,
      { role: 'user', content: transcript }
    ]
  });

  return JSON.parse(response.choices[0].message.content);
}

export async function transcribeAudio(audioFile) {
  return await groq.audio.transcriptions.create({
    file: audioFile,
    model: 'whisper-large-v3-turbo',
    response_format: 'json',
    temperature: 0
  });
}
JS — store/brandscript.store.js 
// Persistent reactive store — survives page refresh
import { reactive, watch } from 'vue'; // or Zustand for React

const STORAGE_KEY = 'vmv8_brandscript';

function loadFromStorage() {
  try {
    const saved = localStorage.getItem(STORAGE_KEY);
    return saved ? JSON.parse(saved) : initEmpty();
  } catch { return initEmpty(); }
}

export const store = reactive({
  brandscript: loadFromStorage(),
  history: [],
  status: 'idle' // idle | transcribing | extracting | done
});

// Auto-save on every change
watch(() => store.brandscript, (val) => {
  localStorage.setItem(STORAGE_KEY, JSON.stringify(val));
}, { deep: true });

JS — pipeline/brandy.pipeline.js 
export async function runBrandyPipeline(audioFile) {
  // Step 1: Transcribe
  store.status = 'transcribing';
  const { text } = await transcribeAudio(audioFile);

  // Step 2: Extract brand strategy as typed JSON
  store.status = 'extracting';
  const extracted = await extractBrandStrategy(text);

  // Step 3: Merge into store (fills only new fields)
  mergeBrandscript(extracted);

  // Step 4: Check completeness — ask follow-up if gaps
  const gaps = findGaps(store.brandscript);
  if (gaps.length > 0) await conductFollowUp(gaps);

  // Step 5: Generate B.I.G Doc
  store.status = 'done';
  return await generateBIGDoc(store.brandscript);
}

import { Document, Packer } from 'docx';

export async function generateBIGDoc(brandscript) {
  const doc = new Document({
    sections: [{
      children: [
        coverPage(brandscript.name),
        ...FRAMEWORK.map(section =>
          renderSection(section, brandscript[section.id])
        )
      ]
    }]
  });

  const buffer = await Packer.toBuffer(doc);

  // Upload to Google Drive automatically
  await uploadToDrive(buffer, `B.I.G Doc — ${brandscript.name.purpose}.docx`);

  // Notify Uvaan
  await sendNotification('B.I.G Doc is ready!');

  return buffer;
}
03 — Build Guide

Implementation Steps

Follow these steps in order. Each step builds on the last. Don't skip the structured output migration — it's the single biggest reliability improvement.
Step 01 — Foundation
Migrate to structured output
Replace all EXTRACT: regex patterns with Groq's JSON mode. Define the BrandScript JSON schema. Delete the dead analyzeInput() function (~150 lines). This is the single most impactful change.
Step 02 — Persistence
Add localStorage store
Wire localStorage save/load into the state object. Progress must survive page refresh. One line saves state; one line restores it on init. Users should never lose a session.
Step 03 — Separation
Split main.js into modules
Extract services/, store/, components/, and config/ into separate files. The 650-line monolith becomes 6 focused modules. Easier to test, easier to hand to another developer.
Step 04 — Safety
Add context window guard
Trim conversationHistory to last 20 messages before every API call. Without this, a long session will eventually hit the model's context limit and fail silently.
Step 05 — Integrations
Google Drive + Canva APIs
Wire Google Drive upload so recordings can be dropped in remotely. Canva API for formatting the B.I.G Doc into a visually branded deliverable. Both use OAuth 2.0.
Step 06 — Types
Add TypeScript
Migrate to TypeScript. Define BrandScript, Section, Field, and Message types. This prevents the silent undefined bugs that come from accessing state with string keys.
